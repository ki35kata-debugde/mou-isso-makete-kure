# データ同期の手順

プレイ本体は **単一HTML**（`mou_isso_v0_6_1.html`）です。  
`data/` 下の JSON を編集しただけではゲームに入らないことがあります。  
種類ごとに同期方法が違います。

---

## いちばん簡単な方法（市場・価格・UI）

1. エクスプローラーで `C:\Users\kzawa\Downloads\Grok\` を開く  
2. **`sync_market.bat`** をダブルクリック  
3. `完了しました` と出たら閉じる  
4. ブラウザで **Ctrl+F5**

反映されるもの:

- 進言費用・買値売値・店在庫・季節補充  
- 完成品売値・馬の★別売値  
- 牧場売却ボタン・進言UI・前期比表示など polish

## 会話・メッセージ（手紙）の同期について

**市場の JSON 同期とは別系統**です。

| データ | 置き場所 | ゲームへの入り方 |
|--------|----------|------------------|
| 手紙本文・選択肢 | `data/letters.json` / `data/game-data.js` | 多くは **HTML 内 `window.GAME_DATA` に埋め込み済み** |
| UI短文 | `data/ui_strings.json` | パッチ適用時に HTML / GAME_DATA へ |
| 馬の名前・★表 | `data/horses.json` → `HORSE_DATA` | 馬パッチ適用時に HTML へ |
| 市場・価格 | `data/market_economy.json` → `MARKET_ECON` | **`sync_market.bat`** |

### 手紙・依頼信頼を直したいとき

1. 手紙: `data/letters.json` または `data/game-data.js` の `letters`  
2. 依頼成功信頼: `data/game-data.js` の  
   - `prep_quests.*.trust_reward`（N0・既定 **4**）  
   - `node_orders.*.1..4.trust_reward`（N1–3=**15**、N4=**20**）  
3. 反映パッチ（例）  
   ```bat
   "C:\Program Files\nodejs\node.exe" data\fix_letters_simplify_and_trust_rewards.js
   ```  
4. **Ctrl+F5**

「市場だけ変えた」のに手紙が古い、というのは **別ファイルだから同期対象外** なのが原因です。  
逆に、手紙用スクリプトは `market_economy.json` を更新しません。

現状、手紙は開発中に HTML へ直接焼き付けている箇所も多いです。  
**確実に変えたい文**がある場合は、どの皇子・どの場面か指定してもらえれば、埋め込み箇所を直す／同期スクリプトを用意できます。

## いつ何が必要か

| 作業 | やること |
|------|----------|
| 価格・進言費・馬売値を変えた | **`sync_market.bat`** |
| 手紙の文章を変えた | 手紙用 `fix_letter_*.js` または HTML 直接（要相談） |
| 仕様書・説明書だけ | 同期不要 |
| HTML を直接いじった | 同期不要（次回 bat で一部上書きされ得る） |

## 注意

- 各スクリプトは実行前に `mou_isso_v0_6_1.html.bak_…` を作ります  
- Node.js が必要（[nodejs.org](https://nodejs.org/) LTS）  
- 反映されない → 開いているファイルパスと **Ctrl+F5** を確認  
- **古いセーブ**には剣×2などが残ることがあります。新規ゲームで初期値を確認してください
