### 各ペアの掛け合い設計

**どのペアも恋愛要素は少なめにする。基本王子が戦争に夢中の様子がコミカルに描かれるが、文通を通じて視野を広げていく様子を書く。**

**牧場の娘 × 馬の王子「文字の少ない、さりげないやさしさ」**
* 前半、二人とも文が短い。王子は寡黙で馬好き。馬に乗っているときが一番解放されて楽しいと感じる

**鍛冶工房の娘 × 兵器の王子「図面で打ち解けていく」**
* 王子：機械の説明になると長い。 → 娘：一目で読み解き、明るい感想と改良案
* 前半、自分の考えた兵器が試したくてしかたがない。攻城戦に使用する兵器のことばかり考えている。
* 中盤、自信家だが周囲に打ち解けられずに悩んでいることが言葉のはしばしに出る。
* 後半、娘の明るさに引っ張られ周囲の兵とも打ち解けていき嬉しそうにする

**市場の娘 × 兵糧の王子「唯一の弱音の宛先」**

* 前半、勝つことが王族として育てられた自分の責務。英才教育により貴族ばかりで兵や農民のことを知らない世間知らずな一面がある。
* 中盤、一緒に戦い、戦が兵の命を犠牲になりたち、兵は兵糧をつくる民にささえていることに気が付き感謝の念を持つ
* 後半、王子の手紙は季節を追って深度を増す：兵を失った夜のこと →「弟たちには弱音を吐けない」

**薬房の娘 × 武器の王子「時差漫才」**

* ボケとツッコミの間に3ヶ月ある漫才。
* 前半、軽口をたたいてばかり、戦の天才。少数精鋭の部隊を率いる。
* 後半軽口の応酬が続いた後、一度だけ届く冗談のない手紙。
「…なあ。俺が誰より早く終わらせれば、皆死なずに済む。そう思うか」

## 4. 市場

### 4.1 経済の考え方

- **同季節の即転売は赤字**（売値 &lt; 買値）  
- 稼ぎは **季節またぎ**（安い季節に買い、高い季節に売る）＋ **完成品・馬の売却**  

### 4.2 価格

```
delta = seasonPriceDelta[今の季節][品目]   # 固定値（両）。無指定は0
買値  = round(baseBuy + delta)
売値  = round(買値 * sellToBuyRatio)  かつ 売 < 買
```

- 買値は **季節×品目ごとの固定差分**（`data/market_economy.json` の `seasonPriceDelta`）で決まる。**店在庫の量には一切左右されない**（旧方式：在庫比率で priceMultMin〜priceMultMax の間を変動、は廃止）  
- 季節番号は 0=春 1=夏 2=秋 3=冬  
- ターン開始: **前期価格保存 → 季節在庫増減（在庫の量のみ。価格には影響しない） → 今季価格再計算**  
- 売買確定後も再計算（在庫が変わっても価格は動かない）  
- 完成品（剣・衝車・回復薬）は従来通り別枠固定（買値999＝店では買えない、売値は`baseSell`）

### 4.3 季節在庫

`seasonStockDelta`（季節0〜3）で増減。`shopMin` で下限。  
詳細キーは [データ同期とメンテナンス.md](./データ同期とメンテナンス.md)。

### 4.4 完成品売却

剣・衝車・回復薬は **売のみ**。価格は `baseSell` 固定寄り。

## 5. 牧場・馬

### 5.1 個体

名前・性・rank（★表示は切り捨て）・速さ・容量・性質・状態  
（`ranch` / `paired` / `transit` / `deployed` 等）

| ★ | 速さ | 容量 |
|---|------|------|
| 1 | 1 | 1 |
| 2 | 2 | 1 |
| 3 | 2 | 2 |
| 4 | 3 | 1 |
| 5 | 3 | 2 |

### 5.2 入手

迷い馬（初回）／名馬探索／繁殖。**金では買えない。**

### 5.3 季節サイクル（配合・確定仕様）

| 季節 | 内容 |
|------|------|
| **春** | **各ペアから仔馬2頭固定**。親は `paired`→`ranch` に復帰しペア解除。冬の餌フラグをリセット。 |
| **夏** | 仔馬が成長（`isFoal`解除・速さ/容量確定）。初年度は出荷解禁の予告。 |
| **秋** | 出荷解禁（初回）。**カップリング期**：成馬オス×メスでペア。枠 `maxPairs`（初期1、厩舎拡張で+1）。**複数ペア可**。**ペア解除して組み直し可**。ペア中は輸送・売却不可。 |
| **冬** | **兵糧のみ**（回復薬は使わない）。牧場にいる全馬（`ranch`/`paired`）各1。与えると**翌春の仔の rank +0.5**（全ペア）。任意。結果ログ：番がいれば「番になった馬に仔馬がうまれそうだ」。 |

### 5.4 rank（潜在値）と表示★

- **表示★** = `floor(rank)`（1〜5）  
- **潜在 rank**: 新規生成時は表示★Nに対し **N.0〜N.9** を振る（例: ★1 → 1.0〜1.9）。旧セーブの整数 `rank=1` も配合時は 1.0〜1.9 帯として扱う  
- **仔馬**: 両親の潜在帯の最小〜最大の間で乱数 → **冬の兵糧で +0.5** → 小数1位、1〜5クランプ  
- 例: ★1×★1＋兵糧 → base 1.0〜1.9 +0.5 → **1.5〜2.4** → 運が良ければ表示★2  

### 5.4b 仔馬の数

- **1ペアにつき2頭固定**

### 5.5 厩舎拡張（進言）

- `maxPairs = 1 + stableExtra`  
- 騎馬次ルート突破で解禁枠が増える  

売却: 出荷解禁後、牧場リストから（`horseSellByStar`）。ペア中・輸送中は不可。

## 6. 輸送

### 6.1 ルート別

| ルート | N0 | 出立後 |
|--------|----|--------|
| 兵糧・兵器・武器 | 手渡し | 人足／荷車／馬車。**船は廃止。馬手段なし** |
| 騎馬次 | 手渡し | **手段なし。馬一覧から選んで自走** |

### 6.2 手段

| 手段 | 解禁 | 速さ | 積載 | 同時使用上限 |
|------|------|------|------|----------------|
| 人足 | 最初から | 1 | 1×人数 | 4人 |
| 荷車 | 兵糧 N1 | 1 | 2 | 50台 |
| 馬車 | 騎馬次 N2 **かつ** 荷車解禁 | 2 | 4 | 100台（利用料50両。船は廃止） |

使用数（人足・車）: `ceil(荷量 ÷ 1単位あたり容量)`。  
出発時に確保、**到着ターンで解放**。

### 6.3 到着時間

```
ETA = ceil( masToNode(route, node) / speed(means) )
```

距離の定義・座標表は [マップ座標の計算.md](./マップ座標の計算.md)。

### 6.4 輸送UI（兵糧など）

- タイトル1行: `ノード1依頼（兵糧0/2個納入）…納期残2T`  
- 品目1行: `兵糧 現庫xx 依頼残xx 輸送中xx` ＋数量  
- 手段選択後: `ノードxまであとxT`  

### 6.5 完納後

1. 信頼加算  
2. `cleared` にノード追加（解禁判定）  
3. `gs.node = node+1`  
4. 新 `order`  
5. **駒を N+1 のマスへ移動**  
6. 手紙用 `pendingEnter`
