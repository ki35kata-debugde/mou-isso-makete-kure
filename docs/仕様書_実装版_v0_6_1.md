## 3. 手紙・信頼・出立・進言

### 3.1 出立前（N0）

| 皇子 | 準備依頼 | 信頼（完了時） |
|------|----------|----------------|
| 兵糧太 | 兵糧×5 | +4 |
| 騎馬次 | 兵糧×4・回復薬×1 | +4 |
| 攻三城 | 衝車★1×2 | +4 |
| 武刃四 | 剣★1×2 | +4 |

- 信頼は `prep_quests.*.trust_reward`（輸送説明文も同値に同期）  
- 出立: 準備完了 ＋ 信頼 ≥ `sortie_trust_threshold`（既定5）  
- **1ターンに出立できるのは1人**  
- 出立ターンは当該ルートへ輸送不可（翌ターンから）  

### 3.2 手紙システム

| 種別 | 誰 | 条件 |
|------|-----|------|
| ノード開始 | 文官（依頼タブ）＋娘 | N0〜N4 進入時 |
| 信頼マイルストーン | 娘 | 信頼 20/40/60/80/100（各 once） |

- 旧: 毎ターン order／催促／雑談フォールバック → **disabled**  
- **新着あり:** タブに赤文字 **`[新着！]`**、選択肢あり  
- **新着なし:** 娘が前回やりとりを再掲。文官もノード文を再掲  
- タブ1行目: `出立待ち` / `戦地壱` / `戦地弐` / `戦地参` / `敵城`  
- **未返信でフェーズ完了:** 確認ダイアログ  
  - OK → 経営へ進む  
  - キャンセル → 手紙に留まる  

状態: `gs.mail[key]`, `gs.letterFlags`（`used`, `pendingEnter`, `justEntered`）。

### 3.3 進言（謁見）

| 項目 | 内容 |
|------|------|
| 費用 | 王子を**褒める**(commend)のみ**300両**（`adviceCostGold`）。他は**150両**（`adviceCostGoldOther`） |
| 回数 | 1ターン1つ。選び直し不可（支払い済み後） |
| 適用 | **今季から即時** |
| 主な効果 | 増産（店在庫≈1年分）、名馬、工房/薬房増員、市場強化、厩舎、褒める(+10) |

解放: 各ルートの `cleared`（N1で1段階、N2で2段階、最大2）。

### 3.4 前線依頼完納の信頼

| ノード | 信頼 |
|--------|------|
| N1〜N3 | +15 |
| N4 | +20 |

`node_orders.*.trust_reward`。完納後 **ノード+1・新依頼・駒移動**。

## 4. 市場

### 4.1 経済の考え方

- **同季節の即転売は赤字**（売値 &lt; 買値）  
- 稼ぎは **季節またぎ**（安い季節に買い、高い季節に売る）＋ **完成品・馬の売却**  

### 4.2 価格

```
delta = seasonPriceDelta[今の季節][品目]   # 固定値（両）。無指定は0
買値  = round(baseBuy + delta)
売値  = round(買値 * sellToBuyRatio)  かつ 売 < 買
```

- 買値は **季節×品目ごとの固定差分**（`data/market_economy.json` の `seasonPriceDelta`）で決まる。**店在庫の量には一切左右されない**（旧方式：在庫比率で priceMultMin〜priceMultMax の間を変動、は廃止）  
- 季節番号は 0=春 1=夏 2=秋 3=冬  
- ターン開始: **前期価格保存 → 季節在庫増減（在庫の量のみ。価格には影響しない） → 今季価格再計算**  
- 売買確定後も再計算（在庫が変わっても価格は動かない）  
- 完成品（剣・衝車・回復薬）は従来通り別枠固定（買値999＝店では買えない、売値は`baseSell`）

### 4.3 季節在庫

`seasonStockDelta`（季節0〜3）で増減。`shopMin` で下限。  
詳細キーは [データ同期とメンテナンス.md](./データ同期とメンテナンス.md)。

### 4.4 完成品売却

剣・衝車・回復薬は **売のみ**。価格は `baseSell` 固定寄り。

## 6. 輸送

### 6.1 ルート別

| ルート | N0 | 出立後 |
|--------|----|--------|
| 兵糧・兵器・武器 | 手渡し | 人足／荷車／船／馬車。**馬手段なし** |
| 騎馬次 | 手渡し | **手段なし。馬一覧から選んで自走** |

### 6.2 手段

| 手段 | 解禁 | 速さ | 積載 | 同時使用上限 |
|------|------|------|------|----------------|
| 人足 | 最初から | 1 | 1×人数 | 4人 |
| 荷車 | 兵糧 N1 | 1 | 2 | 50台 |
| 船 | 兵糧 N2 | 3 | 4 | （台数制限なしの実装） |
| 馬車 | 騎馬次 N2 **かつ** 荷車解禁 | 2 | 2 | 100台 |

使用数（人足・車）: `ceil(荷量 ÷ 1単位あたり容量)`。  
出発時に確保、**到着ターンで解放**。

### 6.3 到着時間

```
ETA = ceil( masToNode(route, node) / speed(means) )
```

距離の定義・座標表は [マップ座標の計算.md](./マップ座標の計算.md)。

### 6.4 輸送UI（兵糧など）

- タイトル1行: `ノード1依頼（兵糧0/2個納入）…納期残2T`  
- 品目1行: `兵糧 現庫xx 依頼残xx 輸送中xx` ＋数量  
- 手段選択後: `ノードxまであとxT`  

### 6.5 完納後

1. 信頼加算  
2. `cleared` にノード追加（解禁判定）  
3. `gs.node = node+1`  
4. 新 `order`  
5. **駒を N+1 のマスへ移動**  
6. 手紙用 `pendingEnter`
