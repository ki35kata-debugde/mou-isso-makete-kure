# データ同期とメンテナンス手順

プレイ本体は **単一 HTML**（`mou_isso_v0_6_1.html`）です。  
`data/` 下の JSON を編集しただけでは反映されないことが多いです。  
**種類ごとに「正本」と「同期方法」が違います。**

---

## 1. まず知っておくこと

| 原則 | 説明 |
|------|------|
| プレイファイル | `mou_isso_v0_6_1.html` をブラウザで開く |
| キャッシュ | 更新後は必ず **Ctrl+F5** |
| バックアップ | 各 `fix_*.js` / `repair_*.js` は実行前に `*.html.bak_*` を作ることが多い |
| Node.js | 同期には必要。標準パス例: `C:\Program Files\nodejs\node.exe` |
| 作業ディレクトリ | 原則 `C:\Users\kzawa\.claude\sessions\Grok_bp` |

---

## 2. データ種別と正本

| 内容 | 正本ファイル | HTML 内の行き先 | 同期方法 |
|------|--------------|-----------------|----------|
| 市場価格・季節在庫・進言費用 | `data/market_economy.json` | `MARKET_ECON = {...}` | `repair_market_embed.js` または `fix_market_season_and_ui.js` / `sync_market.bat` |
| 依頼・信頼報酬・手紙本文 | `data/game-data.js` / `data/letters.json` | `GAME_DATA` / `CFG` | 手紙用 `fix_letters_*.js` または手編集＋埋め込み |
| UI 短文 | `data/ui_strings.json` | `UI_STRINGS` | `sync_ui.js`（古い経路）または直接 HTML |
| 馬名★表 | `data/horses.json` | `HORSE_DATA` | 馬パッチ適用時 |
| マップ座標 | `data/build_map_cells.js` → `map_cells.js` | `MAP_CELLS` | `node data/build_map_cells.js` |
| ロジック変更 | `data/fix_*.js` | HTML 末尾に IIFE 追記 | 各 fix を node で実行 |

## 3. よく使う同期コマンド

### 3.1 市場だけ変えた（いちばん多い）

```bat
cd /d C:\Users\kzawa\.claude\sessions\Grok_bp
"C:\Program Files\nodejs\node.exe" data\repair_market_embed.js
```

またはエクスプローラーで **`sync_market.bat`** をダブルクリック。

その後ブラウザで **Ctrl+F5**。

### 3.2 市場＋季節価格ロジックをまとめて当てる

```bat
"C:\Program Files\nodejs\node.exe" data\fix_market_season_and_ui.js
"C:\Program Files\nodejs\node.exe" data\repair_market_embed.js
```

### 3.3 マップ座標を再計算

```bat
"C:\Program Files\nodejs\node.exe" data\build_map_cells.js
```

→ `data/map_cells.js` 更新。HTML が script src で読めばリロードでOK。

### 3.4 手紙・依頼信頼を大きく変えた

1. `data/letters.json` および/または `data/game-data.js` を編集  
2. 例:
   ```bat
   "C:\Program Files\nodejs\node.exe" data\fix_letters_simplify_and_trust_rewards.js
   ```
3. Ctrl+F5  

※ 手紙は HTML 埋め込みが肥大しているため、**小さな文言修正は HTML 内検索の方が確実な場合**もあります。

### 3.5 ロジックパッチ（機能追加・バグ修正）

例:

```bat
"C:\Program Files\nodejs\node.exe" data\fix_node_advance_piece.js
"C:\Program Files\nodejs\node.exe" data\fix_letter_confirm_proceed.js
"C:\Program Files\nodejs\node.exe" data\fix_eta_winter_tp_ui.js
"C:\Program Files\nodejs\node.exe" data\fix_horse_prince_transport.js
```

パッチは **HTML 末尾にスクリプトを足す**方式が多いです。  
同じマーカー（`[xxx] ready`）がある場合は置き換えます。

### 3.6 壊れたとき・JSON をいじりすぎたとき

```bat
"C:\Program Files\nodejs\node.exe" data\repair_market_embed.js
```

- JSON を正規化して `MARKET_ECON` を再埋め込み  
- 全 `<script>` の構文チェック  

戻し方: Git管理下のため `git checkout -- mou_isso_v0_6_1.html` 等で復元（同フォルダに `.bak_*` が残っていればそちらでも可）。

## 4. JSON 編集の注意

1. **キーは英小文字**（`iron`。`Iron` は無効）  
2. コメント `//` は JSON では不可  
3. 末尾カンマ禁止  
4. 文字コード UTF-8（BOM なし推奨）  
5. 編集後は必ず **同期スクリプト → Ctrl+F5**  
6. セーブ（localStorage）は古い数値のまま残ることがある → 挙動確認は **新規ゲーム** が確実  

### market_economy.json の主要キー

| キー | 説明 |
|------|------|
| `adviceCostGold` | 進言費用（両） |
| `baseBuy` / `baseSell` | 基準買値・売値 |
| `shopStart` / `shopCapBase` | 初期店在庫・上限 |
| `seasonStockDelta` | 季節ごとの店在庫の増減（負可・数量のみ。価格には影響しない） |
| `shopMin` | 店在庫下限 |
| `seasonPriceDelta` | **季節×品目ごとの固定価格差**（両。負可）。買値 = baseBuy + この値。在庫比率とは無関係 |
| `priceMultMin` / `priceMultMax` | 旧・在庫連動の価格倍率。**現在は未使用**（seasonPriceDelta方式に置き換え済み。値は互換のため残置） |
| `sellToBuyRatio` | 買値から売値を出す比率（&lt;1） |
| `horseSellByStar` | 馬売却額 |
| `marketLines` | 市場娘セリフ |

### game-data.js の主要キー

| キー | 説明 |
|------|------|
| `start` | 初期金・所持 |
| `prep_quests` | N0 手渡し依頼・`trust_reward` |
| `node_orders` | N1〜N4 依頼・`trust_reward` |
| `letters` | 手紙プール（条件付き） |
| `recipes_smith` / `recipes_pharmacy` | 生産レシピ |
| `sortie_trust_threshold` | 出立に必要な信頼 |

## 6. よくある失敗

| 症状 | 原因 | 対処 |
|------|------|------|
| 数値を変えたのに変わらない | 同期していない / キャッシュ | 同期スクリプト + Ctrl+F5 |
| SyntaxError | JSON 破損 or 埋め込み破壊 | `repair_market_embed.js` |
| 古いセーブだけおかしい | localStorage | 新規ゲーム or セーブ削除 |
| パッチが効かない | 上書き順で後のパッチが前を潰す | 末尾の `window.xxx =` を確認 |
| 鉄だけ季節変動しない | キー `Iron` 大文字 | `iron` に直す |

| `fix_letters_simplify_and_trust_rewards.js` | 手紙簡素化・信頼報酬 |
| `fix_letter_confirm_proceed.js` | 未返信でも確認で進む |
