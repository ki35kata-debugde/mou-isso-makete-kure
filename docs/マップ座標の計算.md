# マップ座標・マス目の計算方法

> 正本スクリプト: `data/build_map_cells.js`  
> 生成物: `data/map_cells.js`（`window.MAP_CELLS`）  
> プレイ HTML は起動時に `map_cells.js` を読み込むか、同等データが埋め込まれます。

座標系は **SVG viewBox `0 0 1100 1100`**（地図背景と同じ）。  
原点は左上、**x は右方向、y は下方向**です。

---

## 1. データの構造

```js
window.MAP_CELLS = {
  CAP: [500, 498],           // 王都（首都）の中心座標
  ROUTES: { food, horse, siege, weapon },  // ルート定義（入力）
  PORTS: { weapon: [3,8], food: [1,5] },   // 河港セル番号
  SIGN_OVERRIDE: { "horse:4": -1 },        // 特定セルのジグザグ符号強制
  cells: { food: [...], horse: [...], ... } // 計算結果（各マスの x,y）
}
```

### ルート（`ROUTES[key]`）の入力パラメータ

| フィールド | 意味 |
|------------|------|
| `start` | ルート開始点 `[x,y]`（王都寄り） |
| `nodes` | ノード N1〜N4 の座標 `[[x,y], ...]`（4点） |
| `between` | 各区間に挟む **小マスの個数** `[start→N1, N1→N2, N2→N3, N3→N4]` |
| `amp` | 小マスの横ずれ振幅（ピクセル） |
| `color` / `name` / `dirn` | 表示用 |

### 各セル（`cells[route][i]`）

| フィールド | 意味 |
|------------|------|
| `x`, `y` | マス中心座標 |
| `no` | ルート内の通し番号（1始まり）＝**到着までのマス数**にも使用 |
| `node` | `true` なら皇子が駐留するノードマス |
| `nidx` | ノードの 0 始まり index（N1→0, N2→1, …） |

---

## 2. 座標の計算アルゴリズム

`data/build_map_cells.js` の `buildCells` と同等です。

### 手順

1. 折れ線の頂点列を作る  
   `pts = [start, nodes[0], nodes[1], nodes[2], nodes[3]]`（5点）

2. 隣接点 `pts[seg]` → `pts[seg+1]` について  
   - 方向ベクトル `(dx, dy)`  
   - 長さ `L = hypot(dx, dy)`  
   - **法線** `(nx, ny) = (-dy/L, dx/L)`（進行方向の左向き）

3. 区間に小マスを `k = between[seg]` 個置く  
   - パラメータ `t = i / (k + 1)`（`i = 1 … k`）  
   - 線分上の点: `(x0,y0) + t * (dx,dy)`  
   - ジグザグ符号:  
     `base_sign = (seg が偶数) ? +1 : -1`  
     `sign = base_sign * (i が奇数 ? +1 : -1)`  
     ※ `SIGN_OVERRIDE[route+":"+no]` があればそれで上書き（例: 北ルート no=4 は -1）  
   - 最終座標:  
     `x = 線分上x + nx * amp * sign`  
     `y = 線分上y + ny * amp * sign`  
   - 小数第1位に丸め

4. 区間の終点（ノード）をセルとして追加  
   - `node: true`, `nidx: seg`（N1 は seg=0）

### 疑似コード

```text
pts = [start] + nodes
no = 1
for seg in 0 .. 3:
  (x0,y0) = pts[seg], (x1,y1) = pts[seg+1]
  (dx,dy) = (x1-x0, y1-y0)
  L = length, (nx,ny) = 単位法線
  k = between[seg]
  for i in 1 .. k:
    t = i/(k+1)
    sign = ジグザグ(seg,i)  // または SIGN_OVERRIDE
    push 小マス (x0+dx*t + nx*amp*sign, y0+dy*t + ny*amp*sign, no++)
  push ノードマス (x1, y1, no++, node=true, nidx=seg)
```

## 4. 到着ターン（ETA）との関係

```text
ETA = ceil( 距離マス / 手段の速さ )
距離マス = 目標ノードセルの no（masToNode）
```

| 手段 | 速さ（マス/T） | 積載 |
|------|----------------|------|
| 人足 jinput | 1 | 1（空き人数分） |
| 荷車 cart | 1 | 2 |
| 船 boat | 3 | 4 |
| 馬車 wagon | 2 | 2 |

例: 兵糧 N1（no=2）を人足 → `ceil(2/1) = 2T`。

コンソール確認:

```js
debugEta('food', 1, 'jinput')
// または
masToNode('food', 1); etaFor('food', 1, 'jinput')
```

### 輸送中のアイコン位置

`posForConvoy(route, targetNode, eta, etaMax)` が  
王都 → 各セル → 目標ノード の折れ線上を、`進捗 = (etaMax-eta)/etaMax` で線形補間します。

## 5. ノード座標だけ変えたいとき

1. `data/build_map_cells.js` の `ROUTES.*.nodes` / `start` / `between` / `amp` を編集  
2. `node data/build_map_cells.js` で再生成  
3. ブラウザ Ctrl+F5  

ノードの「ゲーム上の意味」（依頼内容）は `game-data.js` の `node_orders` 側です。座標とは別です。
